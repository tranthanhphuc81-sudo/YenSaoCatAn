1. Cấu Trúc Dự Án

Tổng quan

Dự án gồm 3 file HTML độc lập, không cần backend hay build tool:

FileVai tròAi dùng

2. Thiết Lập Firebase

Bước đầu tiên
  1. Truy cập console.firebase.google.com và đăng nhập bằng tài khoản Google.
  2. Nhấn "Add project" → Đặt tên dự án (VD: yen-sao-cat-an) → Tạo dự án.
  3. Trong trang dự án, nhấn biểu tượng </> (Web) để thêm ứng dụng Web.
  4. Đặt tên app, không cần bật Firebase Hosting. Nhấn "Register app".
  5. Sao chép toàn bộ đoạn firebaseConfig (bước tiếp theo hướng dẫn cách dùng).
  6. Vào Build → Firestore Database → Nhấn "Create database" → Chọn Production mode → Chọn vùng gần nhất (VD: asia-southeast1).
  7. Vào Build → Authentication → Tab "Sign-in method" → Bật Email/Password.
⚙️

3. Điền Firebase Config Vào Code

Bắt buộc

Mở cả index.html và admin.html, tìm đoạn sau (xuất hiện trong thẻ <script type="module">) và điền giá trị thực:

// Sao chép từ Firebase Console → Project Settings → Your apps
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "ten-du-an.firebaseapp.com",
  projectId: "ten-du-an",
  storageBucket: "ten-du-an.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123:web:abcdef"
};
⚠️ Cần điền vào cả 2 file giống nhau. Nếu chỉ điền 1 file thì file kia sẽ không kết nối được Firebase.
👤

4. Tạo Tài Khoản Admin

Một lần duy nhất
  1. Vào Firebase Console → Authentication → Users.
  2. Nhấn "Add user".
  3. Điền Email và Mật khẩu cho tài khoản admin (VD: admin@yensaocatan.com).
  4. Nhấn "Add user" để tạo.
  5. Dùng email/mật khẩu này để đăng nhập vào admin.html.
🔒 Đặt mật khẩu mạnh (tối thiểu 8 ký tự, có chữ hoa + số). Không chia sẻ tài khoản này cho khách hàng.

5. Security Rules Firestore

Quan trọng

Vào Firestore → Rules, dán nội dung sau rồi nhấn "Publish":

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Sản phẩm: ai cũng đọc được, chỉ admin mới sửa/xoá
    match /products/{id} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    // Cấu hình: ai cũng đọc được, chỉ admin mới sửa
    match /config/{id} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    // Đơn hàng: ai cũng tạo được, chỉ admin mới xem/sửa/xoá
    match /orders/{id} {
      allow create: if true;
      allow read, update, delete: if request.auth != null;
    }
  }
}

6. Hướng Dẫn Trang Khách Hàng (index.html)

Dành cho người mua

Giao diện mới của trang khách

  • Thanh điều hướng tab nằm ở trên cùng và luôn dính khi cuộn trang.
  • Mỗi tab có icon + nhãn, gồm: Mua sắm, Tin tức, Hướng dẫn, Kiến thức, Lịch sử, Liên hệ.
  • Tab Tin tức chỉ tải dữ liệu khi bạn mở lần đầu, giúp trang tải nhanh hơn.

Quy trình mua hàng

  1. Trang tự động tải ảnh nền và danh sách sản phẩm từ Firebase.
  2. Dùng các nút lọc (Tất cả / Yến Sào / Sâm / Mỹ-Nhật) để tìm sản phẩm.
  3. Click vào thẻ sản phẩm để xem chi tiết trong modal, hoặc nhấn "+ Thêm vào giỏ" trực tiếp.
  4. Kiểm tra giỏ hàng ở phần bên dưới – nhấn ✕ để bỏ sản phẩm.
  5. Điền họ tên, SĐT, địa chỉ và chọn hình thức thanh toán.
  6. Nếu chọn Chuyển khoản, thông tin tài khoản ngân hàng sẽ hiện ra.
  7. Nhấn "Đặt hàng ngay" – đơn hàng được lưu vào Firestore và thông báo thành công.

Tab Tin tức

  • Tab Tin tức hiển thị các bài đăng từ Firestore.
  • Click vào mỗi bài để mở modal đọc chi tiết nội dung, xem ảnh, thẻ tag, ngày đăng và nội dung đầy đủ.

Chat hỗ trợ

  • Ở góc dưới bên phải có nút chat 💬 để mở hộp thoại hỗ trợ.
  • Khi mở lần đầu, bạn cần nhập họ tên và số điện thoại để bắt đầu.
  • Chat cho phép gửi ảnh/video để gửi hình ảnh sản phẩm hoặc yêu cầu cụ thể hơn.
  • Người dùng có thể thu hồi tin nhắn của chính mình bằng nút Thu hồi phía dưới mỗi tin nhắn.
  • Ảnh/video sẽ được upload qua Cloudinary và nén lại trước khi lưu vào cơ sở dữ liệu.
  • Để chạy tính năng media, bạn cần điền giá trị CLOUDINARY_CLOUD_NAME, CLOUDINARY_IMAGE_UPLOAD_PRESET và CLOUDINARY_VIDEO_UPLOAD_PRESET trong index.html.

Lưu ý

  • Số điện thoại phải đúng định dạng Việt Nam (bắt đầu bằng 0 hoặc +84).
  • Giỏ hàng chỉ lưu trong phiên trình duyệt, sẽ mất khi tải lại trang.
  • Có thể thêm cùng một sản phẩm nhiều lần – số lượng sẽ tự cộng dồn.

Tính năng nạp đơn tự động và xác nhận

  • Trang khách có thể nhận đơn từ URL bằng tham số autoOrder=1.
  • Tham số items chứa danh sách identifier:qty, ví dụ items=yen-sao-bac:2,1234567890123:1. Tham số productIds cũng được hỗ trợ như alias.
  • Hệ thống tìm sản phẩm theo ID hoặc mã vạch, nếu không tìm được sẽ thử theo tên.
  • Nếu có autoConfirm=1, hệ thống sẽ mở popup xác nhận trước khi gửi đơn.
  • Popup xác nhận chỉ cho phép đặt hàng khi form thông tin khách hàng đã hợp lệ.

7. Hướng Dẫn Trang Quản Trị (admin.html)

Dành cho admin

Đăng nhập

  1. Mở admin.html trong trình duyệt.
  2. Nhập email và mật khẩu đã tạo ở Bước 4.
  3. Nhấn "Đăng nhập" – Dashboard tự hiển thị.

Tab ⚙️ Cấu Hình

  • Điền URL ảnh để làm nền trang chủ (dùng link ảnh từ Google Drive, Imgur, v.v.).
  • Điền thông tin ngân hàng để hiển thị khi khách chọn chuyển khoản.
  • Điền URL ảnh VietQR để khách quét mã thanh toán.
  • Nhấn "💾 Lưu Cấu Hình" – dữ liệu cập nhật ngay lên Firestore.

Tab 📦 Sản Phẩm

  • Thêm sản phẩm: Điền tên, giá (số nguyên, VNĐ), link ảnh, chọn phân loại rồi nhấn + Thêm sản phẩm.
  • Mã vạch / SKU: Có thể nhập tay hoặc để trống để hệ thống tự tạo mã.
  • Tạo mã QR / in nhãn: Nhấn Tạo mã để xem barcode và QR, sau đó nhấn In nhãn để xuất ra trang in nhiệt.
  • Giấy in 50mm: Trang in đã tối ưu cho máy in nhiệt mini dùng giấy liên tục 50mm.
  • Xoá sản phẩm: Nhấn nút 🗑 Xoá ở hàng tương ứng trong bảng danh sách.
  • Link ảnh nên là URL công khai (không cần đăng nhập để xem).

Tab 📋 Đơn Hàng

  • Hiển thị toàn bộ đơn đã đặt, sắp xếp từ mới nhất đến cũ nhất.
  • Mỗi dòng hiển thị: thời gian, tên khách, SĐT, địa chỉ, tổng tiền, trạng thái, nguồn đơn.
  • Có bộ lọc Tất cả / Tự động / Thủ công để xem nhanh đơn theo nguồn.
  • Nhấn Xem chi tiết để mở modal xem rõ danh sách sản phẩm, trạng thái, thanh toán và nguồn.

Đăng xuất

Nhấn nút "Đăng xuất" ở góc trên bên phải. Trang sẽ trở về form đăng nhập.

8. Hướng Dẫn Trang Kho Hàng (inventory.html)

Dành cho quản lý kho

Chức năng chính

  • Nhập mã vạch hoặc quét mã bằng camera để tìm sản phẩm trong kho.
  • Hệ thống đọc mã vạch và tự động nạp sản phẩm lên form nếu đã có trong products.
  • Điều chỉnh tồn kho bằng Nhập hoặc Xuất với số lượng và ghi chú.
  • Nhấn Lưu tồn kho để cập nhật số lượng trên Firestore và đồng thời ghi lại bản ghi vào stockMovements.
  • Danh sách sản phẩm và lịch sử nhập xuất được tải từ Firestore; có thể tìm theo mã vạch, tên, vị trí kho.

Quy trình vận hành

  1. Nhập hoặc quét mã vạch trên sản phẩm.
  2. Hệ thống tìm sản phẩm trong collection products theo trường barcode.
  3. Nếu tìm thấy, form sẽ hiển thị tên, ảnh, tồn kho hiện tại để admin chỉnh sửa.
  4. Chọn loại thao tác Nhập hoặc Xuất, nhập số lượng và ghi chú.
  5. Nhấn Lưu tồn kho để cập nhật products.quantity và tạo bản ghi mới trong stockMovements.
ℹ️ Nếu sản phẩm chưa tồn tại trên admin, hệ thống sẽ yêu cầu tạo sản phẩm trước rồi mới cập nhật tồn kho.

9. Triển Khai Lên GitHub Pages

Xuất bản
  1. Tạo repository mới trên github.com (VD: yen-sao-cat-an).
  2. Upload 3 file index.html, admin.html, huong-dan.html lên repo.
  3. Vào tab Settings → Pages → Source: chọn Deploy from a branch, Branch: main, thư mục: / (root) → Nhấn Save.
  4. Sau vài phút, trang sẽ live tại: https://username.github.io/yen-sao-cat-an/
  5. Vào Firebase Console → Authentication → Settings → Authorized domains → Thêm domain GitHub Pages vừa có.
✅ GitHub Pages hoàn toàn miễn phí với repository public. Firebase Spark (free tier) đủ dùng cho quy mô nhỏ.
⚠️ Nếu không thêm domain GitHub Pages vào Authorized domains, Firebase Authentication sẽ báo lỗi khi đăng nhập từ URL đó.

11. Xử Lý Sự Cố Thường Gặp

12. Luồng Hoạt Động & Kiểm Soát

Kiến trúc

Chủ thể và nhiệm vụ

  • Khách hàng: duyệt sản phẩm, nạp giỏ, điền thông tin, đặt hàng.
  • Admin: quản lý sản phẩm, xem đơn, lọc đơn theo auto hoặc manual, điều chỉnh tồn kho.
  • Firestore: lưu trữ cấu hình, sản phẩm, đơn hàng và lịch sử nhập xuất.

Ảnh hưởng và kiểm soát

  • Order source: đơn tự động được gán source: 'auto', đơn website thông thường gán source: 'website'.
  • Kiểm soát admin: chỉ admin đăng nhập mới đọc/sửa dữ liệu
    trong admin.html, theo rules đã đặt trong Firestore.
  • Kiểm soát tồn kho: mỗi khi đặt hàng, hệ thống trừ products.quantity và ghi vào stockMovements.
  • Xác nhận đơn tự động: popup chỉ cho phép gửi khi dữ liệu khách hàng hợp lệ, tránh đơn thiếu thông tin.

Sơ đồ luồng hoạt động

// Mermaid syntax cho sơ đồ hoạt động
graph LR
  KH[Khách hàng] -->|Xem sản phẩm| WEB[index.html]
  WEB -->|Đặt hàng| FIRESTORE[Firestore orders]
  WEB -->|Nạp đơn tự động| WEB
  WEB -->|Kiểm tra| STOCK[products.quantity]
  FIRESTORE -->|Ghi order| ORD[orders collection]
  FIRESTORE -->|Cập nhật| STOCK
  ADMIN[Admin] -->|Xem/lọc đơn| ORD
  ADMIN -->|Quản lý sản phẩm| PRODUCTS
  ADMIN -->|Quản lý kho| STOCKMOV[stockMovements collection]
✅ Đây là sơ đồ đơn giản thể hiện 3 chủ thể chính: Khách hàng, Admin, Firestore. Mỗi hành động đều ảnh hưởng đến dữ liệu và được kiểm soát qua phép viết đọc của Firestore.
